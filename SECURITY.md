# Security Policy

## Supported Versions

Security fixes are considered for the latest released version.

## Reporting a Vulnerability

Please report security issues privately to the project maintainers. Do not open a public issue containing sensitive logs, audio captures, hostnames, usernames, device identifiers, or other private environment details.

## Data Handling

This project is intended to run locally on files you provide. It does not send logs or audio files to a network service.

When sharing examples or bug reports, use synthetic data or redact:

- Usernames and machine names.
- Audio device identifiers.
- Paths containing personal information.
- Captured audio content.
- Application-specific secrets or tokens.
